# Feature 003. FAQ, информационные страницы и документы: критерии приемки

## Общие критерии
- Публичная навигация содержит переходы к FAQ, информационным страницам и документам.
- Маршруты `/FAQ`, `/faq`, `/info/:section?` и `/documents/:documentType` открываются в публичной оболочке без перезагрузки приложения.
- Для неизвестного FAQ-фильтра, информационного раздела или типа документа пользователь видит локализованное состояние недоступности, а backend не раскрывает несуществующие материалы.
- Все пользовательские тексты на frontend берутся из i18n-словарей на русском и английском языках.
- Backend возвращает только i18n-ключи и mnemonic-коды `STR_MNEMO_*` для предопределенных пользовательских сообщений.

## FAQ
- Пользователь видит список категорий FAQ, количество вопросов в категории и общий список опубликованных вопросов.
- Пользователь может отфильтровать FAQ по категории.
- Пользователь может выполнить поиск по вопросу, ответу и тегам; результаты обновляются без потери выбранной роли пользователя.
- Каждый FAQ-ответ может содержать ссылку на связанный информационный раздел или документ.
- При отсутствии результатов backend возвращает пустой список и код `STR_MNEMO_PUBLIC_FAQ_EMPTY`, а frontend показывает локализованное сообщение.

## Информационные разделы
- Маршрут `/info/:section` показывает заголовок, описание, хлебные крошки, SEO-метаданные и отсортированные rich text блоки.
- Маршрут `/info` открывает обзорный раздел с доступными публичными темами.
- Информационный раздел может содержать вложенные якоря, CTA и связанные документы.
- Ролевые ограничения не скрывают базовые публичные разделы для гостя, но могут добавлять партнерские материалы авторизованному партнеру.

## Документы
- Маршрут `/documents/:documentType` показывает список документов выбранного типа с названием, версией, датой публикации, статусом актуальности, PDF viewer URL и ссылкой скачивания.
- Для каждого документа доступен архив предыдущих редакций без замены актуального документа.
- Документы фильтруются по аудитории: гостю доступны публичные документы, клиенту и партнеру дополнительно доступны релевантные материалы их статуса.
- Обязательные документы явно помечаются признаком `required=true`.

## Интеграция и качество
- API-тесты покрывают FAQ, информационный раздел, документы, ролевую фильтрацию и недоступный материал.
- UI-тесты покрывают открытие FAQ, поиск, переход к информационному разделу, отображение PDF-документа и fallback для неизвестного типа.
- Managed end-to-end тесты агрегируют реальные feature-тесты feature #1, #2 и #3 без placeholder-проверок.
