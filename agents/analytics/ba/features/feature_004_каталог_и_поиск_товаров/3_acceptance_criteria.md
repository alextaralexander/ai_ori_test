# Feature 004. Каталог и поиск товаров: критерии приемки

## Общие критерии
- Маршрут `/search` открывается в публичной оболочке без перезагрузки приложения для гостя, покупателя и партнера.
- Поисковая строка принимает название, код, категорию и ключевые слова товара; результат зависит от опубликованных товаров текущей кампании.
- Параметры `q`, `category`, `priceMin`, `priceMax`, `availability`, `tags`, `promo`, `sort` и `page` отражаются в URL и восстанавливают состояние выдачи при прямом открытии ссылки.
- Все пользовательские тексты на frontend берутся из i18n-словарей на русском и английском языках.
- Backend возвращает только данные и mnemonic-коды `STR_MNEMO_*` для предопределенных пользовательских сообщений, которые могут попасть во frontend.
- Каталог не показывает товары, которые не опубликованы, не относятся к активной кампании или запрещены для текущей роли.

## Поиск и выдача
- Поиск по пустому запросу показывает первую страницу каталога с сортировкой по релевантности кампании.
- Поиск по части названия, артикулу, категории или тегу возвращает релевантные опубликованные товары.
- Каждый результат содержит идентификатор товара, артикул, slug, название, категорию, изображение, краткое описание, цену, валюту, доступность, теги, промо-метки и признак возможности добавления в корзину.
- Гость видит только публичные цены и публичную доступность; покупатель и партнер видят условия, доступные для своей роли.
- При отсутствии результатов backend возвращает пустой список, рекомендации и код `STR_MNEMO_CATALOG_SEARCH_EMPTY`, а frontend показывает локализованное пустое состояние.

## Фильтры, сортировка и пагинация
- Категории, диапазон цены, доступность, теги и промо-метки применяются совместно и не сбрасывают поисковый запрос.
- Сортировка поддерживает значения `relevance`, `newest`, `priceAsc`, `priceDesc` и `popular`.
- Пагинация возвращает номер страницы, размер страницы, общее количество найденных товаров и признак наличия следующей страницы.
- Lazy loading следующей страницы не дублирует уже показанные товары и сохраняет выбранные фильтры.
- Некорректные значения фильтров нормализуются к безопасным значениям или игнорируются без раскрытия технической ошибки.

## Добавление в корзину
- Для доступного товара авторизованный покупатель или партнер может выполнить добавление в корзину из списка.
- Для гостя кнопка добавления в корзину переводит пользователя к сценарию входа или регистрации без потери текущего URL поиска.
- Если товар недоступен, кнопка добавления заблокирована, а карточка показывает локализованную причину через i18n.
- После успешного добавления frontend получает обновленное краткое состояние корзины и показывает локализованное подтверждение.
- При конфликте остатка или кампании backend возвращает mnemonic-код `STR_MNEMO_CATALOG_ITEM_UNAVAILABLE`.

## Интеграция и качество
- API-тесты покрывают поиск по тексту, фильтры, сортировки, пагинацию, пустую выдачу, ролевую видимость и добавление товара в корзину.
- UI-тесты покрывают открытие `/search`, ввод запроса, применение фильтров, сортировку, lazy loading, пустую выдачу и добавление товара в корзину авторизованным пользователем.
- Managed end-to-end тесты агрегируют реальные feature-тесты feature #1, #2, #3 и #4 без placeholder-проверок.
- Реализация не нарушает публичную навигацию, новости, FAQ, информационные страницы и документы из предыдущих фич.

## Версионная база
Новые runtime-технологии не добавляются. Для совместимости с уже реализованными feature #1-#3 используется текущий backend monolith и frontend web-shell baseline репозитория; возможное обновление до новых стабильных major/minor версий должно быть вынесено в отдельную техническую задачу с миграционными тестами.
