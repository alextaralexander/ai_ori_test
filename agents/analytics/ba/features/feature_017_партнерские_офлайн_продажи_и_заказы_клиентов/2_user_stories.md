# User stories feature 017. Партнерские офлайн-продажи и заказы клиентов

## Контекст
Фича добавляет партнерский контур офлайн-продаж Best Ori Gin: партнер видит заказы клиентов, оформленные через него вне прямого self-service сценария, сопровождает оплату, доставку, повторный заказ и сервисные корректировки. Базовый технологический baseline задачи: Java 25 как текущий утвержденный runtime репозитория при наличии Java 26 GA на дату старта, Spring Boot 4.0.6, Maven, React 19.2, TypeScript 6.0, Ant Design latest через web-shell; новых runtime-технологий фича не вводит.

## Роли
- Партнер-лидер: контролирует клиентские и партнерские заказы своей структуры, оценивает бонусный и MLM-эффект, помогает с повторными заказами.
- Партнер-продавец: оформляет и сопровождает офлайн-заказы своих клиентов, проверяет оплату, состав, доставку и доступные действия.
- Сотрудник поддержки заказов: видит заказы клиентов партнера, выполняет разрешенные сервисные корректировки и фиксирует аудит действий.

## Stories
### US-017-01. Список VIP-заказов партнера
Как партнер-продавец, я хочу открыть `/vip-orders` и увидеть список клиентских заказов, связанных со мной, чтобы быстро понять статус оплаты, доставки и сумму заказа.

### US-017-02. Управление заказами клиентов в бизнес-инструментах
Как партнер-лидер, я хочу открыть `/business/tools/order-management/vip-orders/partner-orders` и фильтровать клиентские и партнерские заказы по каталогу, статусу, клиенту и номеру заказа, чтобы управлять офлайн-продажами в текущей кампании.

### US-017-03. Детальная карточка заказа клиента
Как партнер-продавец, я хочу открыть `/business/tools/order-management/vip-orders/partner-orders/:orderId`, чтобы увидеть состав заказа, оплату, доставку, бонусные начисления, историю событий и связанные действия.

### US-017-04. Сервисные действия по заказу
Как сотрудник поддержки заказов, я хочу выполнять разрешенные корректировки по офлайн-заказу, чтобы инициировать повторный заказ, зафиксировать сервисный комментарий или отметить корректировку без раскрытия лишних данных.

### US-017-05. Связка с MLM и отчетами партнера
Как партнер-лидер, я хочу видеть personNumber партнера, customerId клиента, businessVolume и bonusAccrualStatus, чтобы понимать влияние офлайн-продажи на отчеты, бонусный кошелек и рост структуры.

### US-017-06. Безопасность и локализация
Как пользователь платформы, я хочу видеть только разрешенные заказы и локализованные сообщения, чтобы backend передавал predefined-сообщения только mnemonic-кодами `STR_MNEMO_*`, а frontend показывал текст из i18n-словарей.
