# Feature 020. Сотрудник: история заказов

## Назначение
Повторить employee order-history с операторским доступом к списку заказов и деталям конкретного заказа.

## Покрываемые маршруты
- `/employee/order-history`
- `/employee/order-history/:orderId`

## Основной функционал
- Список заказов в операторском представлении.
- Фильтры по партнеру, периоду, статусу и проблемным кейсам.
- Детали заказа с более широким набором сервисных данных.
- Переходы в support, claims и платежные события.
- Логирование просмотра и действий сотрудника.

## Роли
- Сотрудник backoffice
- Супервизор
